# 11 — UI and UX Design

## UX Principle

The user should interact with findings, evidence, patterns, and recommendations — not raw prompts.

## Primary User Flow

```text
1. Upload or paste transcript
2. Confirm speakers
3. Choose analysis type
4. Review analysis scope
5. Run workflow
6. Explore report dashboard
7. Drill into evidence and module details
8. Export or save report
```

## Transcript Upload Screen

Elements:

- Paste transcript box
- File upload
- Speaker labeling helper
- Privacy notice
- Transcript quality indicator

## Analysis Selection Screen

Show user-facing suites:

- Quick Review
- Conflict Coaching
- Mediation-Oriented
- Clinical Exploration
- Full Multidisciplinary Suite

Each should show:

- estimated depth
- included lenses
- best for
- output type

## Full Suite Explanation

User-facing language:

```text
This analysis examines the transcript through communication, cognitive, systems, mediation, Gottman, NVC, attachment-informed, trauma-informed, needs/values, narrative, and evidence-quality lenses. It then synthesizes the findings into an integrated report.
```

## Report Dashboard

Top-level sections:

1. Executive Summary
2. Primary Interaction Pattern
3. Strongest Findings
4. Biggest Uncertainties
5. Highest-Leverage Improvements
6. Evidence Map
7. Module Reports
8. Integrated Synthesis

## Finding Card Design

Each card should include:

- title
- summary
- confidence badge
- supporting evidence quotes
- source modules
- alternative explanations
- limitations
- recommended action, if applicable

## Evidence Viewer

Clicking a quote ID should reveal:

- exact quote
- speaker
- turn number
- surrounding context
- modules that used it

## Module Tabs

Tabs should include:

- Communication
- Cognitive
- Systems
- Mediation
- Gottman
- NVC
- Attachment
- Trauma-Informed
- Needs & Values
- Narrative
- Bias Audit
- Synthesis

## Interaction Cycle Visualization

Use simple diagrams:

```text
Concern raised
  ↓
Explanation/defense
  ↓
Feeling unheard
  ↓
Escalation
  ↓
Withdrawal/counterargument
```

## Confidence UI

Use clear labels:

- Directly observed
- Strongly supported
- Reasonable interpretation
- Tentative possibility
- Exploratory hypothesis
- Not enough evidence

Avoid technical jargon.

## Progressive Disclosure

Do not show everything at once.

Start with:

- what matters most
- confidence
- next steps

Allow deeper exploration through expandable sections.

## Export Formats

Later versions should export:

- Markdown
- PDF
- therapist-ready report
- mediation brief
- coaching action plan
- JSON data export

## Safety UX

Include reminders:

- This is not a diagnosis
- This does not determine abuse or legal responsibility
- Consider professional support for serious safety concerns
- Transcript context may be incomplete
