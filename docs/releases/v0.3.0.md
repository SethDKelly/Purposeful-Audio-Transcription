## RRE v0.3.0 — Post-MVP expansion (Phases H–L)

Major release after the v0.2.0 MVP. Delivers legacy cleanup, full module/workflow library, exports, platform hardening, and interactive exploration.

### Highlights

**Phase H — Legacy cleanup**
- Workflow-only analysis path; removed `POST /api/analyze` and purpose registry
- `POST /api/modules/{id}/stream` for single-module streaming
- `GET /api/purposes` deprecated alias over `/api/modules`
- Streamlit workflow-only UI

**Phase I — Module & workflow expansion**
- **13 analysis modules** (full prompt library)
- **5 workflows:** `quick_review`, `full_mvp`, `conflict_coaching`, `mediation_brief`, `clinical_exploration`
- Workflow metadata: `output_tone`, `recommended_model`

**Phase J — Export & reporting**
- PDF workflow reports (`fpdf2`)
- Coach summary and mediation brief markdown exports
- Optional redaction before export

**Phase K — Platform hardening**
- PostgreSQL option with connection pooling (`DATABASE_URL`)
- Alembic migrations (`alembic upgrade head` or `ALEMBIC_AUTO_UPGRADE=true`)
- Background workflow jobs with resume on API restart
- Optional `X-API-Key` authentication
- Structured JSON logging and workflow telemetry

**Phase L — Interactive exploration**
- Finding drill-down with evidence chains
- Cross-module agreement/tension explorer
- Compare workflow runs across sessions
- Knowledge graph from module constructs
- Scoped follow-up Q&A without re-running workflows
- Streamlit **Explore** tab + REST APIs for future React UI

### Breaking changes

- `POST /api/analyze` removed — use `POST /api/workflows/{id}/run` or `POST /api/modules/{id}/stream`
- `config/purposes.yaml` archived to `config/archived/purposes.yaml`
- Streamlit no longer supports legacy single-purpose mode

### New configuration

| Variable | Default | Purpose |
|----------|---------|---------|
| `ALEMBIC_AUTO_UPGRADE` | `false` | Run migrations on startup (PostgreSQL) |
| `API_KEY` | _(empty)_ | Require `X-API-Key` on protected routes |
| `LOG_JSON` | `false` | Structured JSON logs |
| `WORKFLOW_BACKGROUND_DEFAULT` | `false` | Queue workflows in background |
| `DATABASE_POOL_SIZE` | `5` | PostgreSQL pool size |
