## RRE v0.6.0 — Trust, workflows, AWS-only prune

Canonical release on `main` after Tier 2 closeout: data-handling trust controls, full multidisciplinary + research workflows, AWS-only product path, Bedrock reliability hardening, and workflow performance for live UAT.

### Highlights

**Trust (P1-3)**
- Temp audio cleanup, cascade transcript DELETE, log redaction, retention, audit trail

**Workflows (P1-4)**
- `full_multidisciplinary` and `research_oriented` workflows
- Default background for long suites; sync module limit + UI warning
- Live burn-in / component UAT on AWS (Bedrock); fuller UAT deferred to next release

**AWS-only prune (P1-7)**
- Removed Ollama / Whisper product dual-path
- Laptop loop: edit → pytest (SQLite) → Deploy to AWS

**Reliability & deploy**
- JSON-only Bedrock prompts + structured output; safer meta-synthesis parse
- API `findings` rollup for clients; burn-in asserts non-empty synthesis
- 2 uvicorn workers; softer ALB drain; clear-before-deploy; ECS steady wait
- Pre-commit Tier 1–2 gates + config validators in Deploy CI

**Workflow speed (without raising model spend)**
- Bounded parallel transcript modules (`WORKFLOW_MODULE_CONCURRENCY`, default 3)
- Leaner module outputs + fewer retries
- Bedrock prompt caching on shared framework + evidence prefix

**UX**
- Workflow poll every 30s; status copy maps `running_modules` → “running workflows”
