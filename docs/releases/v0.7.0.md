# v0.7.0 — Trust, transcript prep, ontology foundation

**Released:** 2026-07-16  
**Theme:** Secure UAT readiness, transcript preparation workspace, ontology vocabulary, construct expectations, golden evaluation foundation, and module/workflow telemetry.

## Highlights

| Area | What shipped |
|------|----------------|
| **Secure UAT** | Shared API key (UI + API), generic errors + `request_id`, privacy/retention captions, export redaction default-on, Terraform ready for optional ACM HTTPS |
| **Transcript prep** | Edit/exclude turns, rebuild evidence index, `analysis_ready` gate before workflows |
| **Ontology v1** | `config/ontology/` construct + relationship catalogs; `OntologyRegistry`; module `expected_constructs` |
| **Construct coverage** | Soft validation warnings (warn, don’t fail); visible on module runs |
| **Golden fixtures** | GT001/GT002 under `tests/fixtures/golden_transcripts/` with signal-based assertions + pytest suite |
| **Telemetry** | Per-module latency/tokens/cost; workflow `telemetry_summary` |
| **Deploy policy** | Deploy on `v*.*.*` tags or manual workflow only — not every `main` push |
| **Docs** | Folder renamed `doc/` → `docs/`; guides updated for v0.7 |

## Operators

1. Deploy: GitHub Actions **Deploy to AWS dev** (or push tag `v0.7.0`).
2. Open ALB UI → ingest → **Prepare** → **Ready to Analyze** → Quick Review.
3. When idle: **Pause AWS dev**.

See [../user/getting-started.md](../user/getting-started.md) · [../developer/aws-operations.md](../developer/aws-operations.md).

## Breaking / behavior notes

- Workflows will not run until a transcript is marked ready (unless skip-review / pytest auto-ready).
- Unknown `expected_constructs` / `required_constructs` in module YAML fail registry load (must resolve via ontology).
- Auto-deploy on ordinary `main` pushes is disabled.
